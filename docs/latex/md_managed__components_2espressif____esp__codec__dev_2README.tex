{\ttfamily esp\+\_\+codec\+\_\+dev} is a driver component for audio codec devices. Following features are supported currently\+:
\begin{DoxyItemize}
\item Support driver for common audio codec devices
\item Support multiple instance of codec devices (including device of same type)
\item Add unified abstract interface to operate on codec device
\item Support customized codec realization based on provided interface
\item Easy-\/to-\/use high-\/level API for playback and recording
\item Support ADC microphone capture through internal ADC data interface ({\ttfamily audio\+\_\+codec\+\_\+new\+\_\+adc\+\_\+data})
\item Support dummy codec for PA-\/only speaker designs ({\ttfamily dummy\+\_\+codec\+\_\+new}), enabling automatic PA control via {\ttfamily esp\+\_\+codec\+\_\+dev\+\_\+open/close}
\item Support for volume adjustment in software when it is not supported in hardware
\item Support customized volume curve and customized volume control
\item Easy to port to other platform after replacing codes under \href{./platform}{\texttt{ platform}}
\end{DoxyItemize}

ESP32-\/\+S3 sends control data to ES8311 through I2C bus and exchanges audio data through I2S bus. During playing, ES8311 receives digital audio data from I2S bus and performs DAC operation, then the analog signal is amplified by PA chip (NS4150) and finally is output through the speaker. During recording, ES8311 gets the analog signal from the microphone, amplifies it, and performs ADC operation, then digital audio data can be obtained from ESP32-\/\+S3.

Communication between ESP32-\/\+S3 and ES8311 mainly occurs on two paths\+:
\begin{DoxyItemize}
\item Control path\+: Set up the codec chip (using I2C bus)
\item Data path\+: Exchange audio data (using I2S bus)
\end{DoxyItemize}

{\ttfamily esp\+\_\+codec\+\_\+dev} abstracts the above communication path into two interfaces\+: ~\newline

\begin{DoxyItemize}
\item {\ttfamily \doxylink{structaudio__codec__ctrl__if__t}{audio\+\_\+codec\+\_\+ctrl\+\_\+if\+\_\+t}} for the control path\+: ~\newline
 The control interface mainly offers {\ttfamily read\+\_\+reg} and {\ttfamily write\+\_\+reg} APIs to do codec setup ~\newline
 Commonly used control channels include I2C, SPI, etc
\item {\ttfamily \doxylink{structaudio__codec__data__if__t}{audio\+\_\+codec\+\_\+data\+\_\+if\+\_\+t}} for data path\+: ~\newline
 The data interface mainly offers {\ttfamily read} and {\ttfamily write} APIs to exchange audio data ~\newline
 Commonly used data channels include I2S, SPI, etc
\end{DoxyItemize}

{\ttfamily esp\+\_\+codec\+\_\+dev} provides users with convenient high-\/level API to implement playback and recording functions. It is composed of {\ttfamily \doxylink{structaudio__codec__data__if__t}{audio\+\_\+codec\+\_\+data\+\_\+if\+\_\+t}} and {\ttfamily \doxylink{structaudio__codec__if__t}{audio\+\_\+codec\+\_\+if\+\_\+t}}. {\ttfamily \doxylink{structaudio__codec__if__t}{audio\+\_\+codec\+\_\+if\+\_\+t}} abstracts codec control operations and constructed by specified codec configuration (configured by {\ttfamily \doxylink{structaudio__codec__ctrl__if__t}{audio\+\_\+codec\+\_\+ctrl\+\_\+if\+\_\+t}} and {\ttfamily \doxylink{structaudio__codec__gpio__if__t}{audio\+\_\+codec\+\_\+gpio\+\_\+if\+\_\+t}} through {\ttfamily \doxylink{structes8311__codec__cfg__t}{es8311\+\_\+codec\+\_\+cfg\+\_\+t}}). {\ttfamily \doxylink{structaudio__codec__gpio__if__t}{audio\+\_\+codec\+\_\+gpio\+\_\+if\+\_\+t}} abstracts the IO control to adapt to the main control IO or the expansion chip IO, and called inside the codec to match the unique set timing.\hypertarget{md_managed__components_2espressif____esp__codec__dev_2README_autotoc_md269}{}\doxysection{\texorpdfstring{DAC Volume setting}{DAC Volume setting}}\label{md_managed__components_2espressif____esp__codec__dev_2README_autotoc_md269}
Volume setting is realized by common API\+: {\ttfamily esp\+\_\+codec\+\_\+dev\+\_\+set\+\_\+out\+\_\+vol}. ~\newline
 {\ttfamily esp\+\_\+codec\+\_\+dev} supports the following volume setup methods\+:
\begin{DoxyEnumerate}
\item Use codec register to adjust the volume
\item Use built-\/in software volume {\ttfamily audio\+\_\+codec\+\_\+new\+\_\+sw\+\_\+vol} when codec hardware does not support volume adjustment
\item Use customized software volume interface through {\ttfamily esp\+\_\+codec\+\_\+dev\+\_\+set\+\_\+vol\+\_\+handler}
\end{DoxyEnumerate}

The default volume range is 0 -\/ 100. Volume \mbox{[}1\+:100\mbox{]} is mapped to \mbox{[}-\/49.\+5 dB\+:0 dB\mbox{]} with the scale being 0.\+5 dB. Volume 0 is mapped to -\/96 dB. To change this mapping, you can define your own volume curve through API {\ttfamily esp\+\_\+codec\+\_\+dev\+\_\+set\+\_\+vol\+\_\+curve}. The volume curve is an array of {\ttfamily \doxylink{structesp__codec__dev__vol__map__t}{esp\+\_\+codec\+\_\+dev\+\_\+vol\+\_\+map\+\_\+t}} which uses linear interpolation to calculate the decibel value at a certain volume point internally (please sort volume maps in advance). ~\newline
 To balance the speaker\textquotesingle{}s loudness across different platforms when playing the same content, you need to know some mechanism of the audio gain. In short, audio gain consists of two parts\+: software gain (adjustable) and hardware gain (fixed). Software gain can be adjusted by changing the input PCM data level or setting the codec volume register. The hardware gain is affected by the peripheral circuit, mainly by the amplification efficiency of the analog signal. The typical impact parameter of hardware gain is extracted into {\ttfamily \doxylink{structesp__codec__dev__hw__gain__t}{esp\+\_\+codec\+\_\+dev\+\_\+hw\+\_\+gain\+\_\+t}}, which can be configured to codec devices to ensure loudness consistency. For more details, please refer to the comments in \href{include/esp_codec_dev_vol.h}{\texttt{ esp\+\_\+codec\+\_\+dev\+\_\+vol.\+h}}.\hypertarget{md_managed__components_2espressif____esp__codec__dev_2README_autotoc_md270}{}\doxysection{\texorpdfstring{Usage}{Usage}}\label{md_managed__components_2espressif____esp__codec__dev_2README_autotoc_md270}
