 {\ttfamily Sensor hub} is a sensor management component that can realize hardware abstraction, device management and data distribution for sensor devices. From the point of application development, this component has the following features\+:


\begin{DoxyEnumerate}
\item Support for registering sensor drivers through component approach, enabling users to focus on usage without concerning themselves with the implementation of the driver layer.
\item Provides a unified sensor management and message notification mechanism, allowing users to simply select sensor operation modes, sampling intervals, ranges, etc., and register callback functions for events of interest. This ensures users are notified when the sensor state changes or data collection is complete.
\end{DoxyEnumerate}

{\ttfamily Sensor hub} can be used as a basic component for sensor applications in various intelligent scenarios such as environment monitoring, motion detection, health management and etc. as it simplifies operation and improves operating efficiency by centralized management of sensors.\hypertarget{md_managed__components_2espressif____sensor__hub_2README_autotoc_md457}{}\doxysection{\texorpdfstring{How to register sensor\+\_\+hub in other sensor drivers}{How to register sensor\_hub in other sensor drivers}}\label{md_managed__components_2espressif____sensor__hub_2README_autotoc_md457}
To enable the {\ttfamily sensor\+\_\+hub} to locate the sensor component, please refer to the following procedure\+:


\begin{DoxyEnumerate}
\item Add a dependency on {\ttfamily sensor\+\_\+hub} in the {\ttfamily idf\+\_\+component.\+yml} file of the sensor driver.
\item Implement specific functions based on the sensor type and register them in the {\ttfamily sensor\+\_\+hub}.
\end{DoxyEnumerate}
