{\ttfamily esp\+\_\+codec\+\_\+dev} 是为音频编解码器设备提供驱动的组件，目前支持以下功能\+:
\begin{DoxyItemize}
\item 提供常用音频编解码器设备的驱动
\item 支持音频编解码器设备的多实例 （包括同类型设备）
\item 提供统一的抽象化接口来操作编解码器设备
\item 支持客户定制化编解码器设备 (仅需实例化提供的接口)
\item 为播放和录音提供易用的上层 API
\item 支持软件音量调节 (硬件不支持音量调节时)
\item 支持定制化音量曲线以及音量控制实现
\item 兼容多平台仅需替换 \href{./platform}{\texttt{ platform}}
\end{DoxyItemize}

ESP32-\/\+S3 通过 I2C 总线向 ES8311 发送控制命令，通过 I2S 总线传递音频数据。在播放过程中, ES8311 从 I2S 总线接收数字音频数据进行数模转换后发送给功放芯片(\+NS4150), 最后发送给扬声器输出声音。在录音过程中，\+ES8311将从麦克风采集到的模拟信号放大，进行模数转换后发送给 ESP32-\/\+S3。 ESP32-\/\+S3 同 ES8311 在以下两个通道进行通讯\+:
\begin{DoxyEnumerate}
\item 控制通道：用来配置编解码器设备 (通过 I2C 总线)
\item 数据通道\+: 用来交换音频数据 (通过 I2S 总线)
\end{DoxyEnumerate}

通讯通道抽象为两种接口\+:
\begin{DoxyItemize}
\item {\ttfamily \doxylink{structaudio__codec__ctrl__if__t}{audio\+\_\+codec\+\_\+ctrl\+\_\+if\+\_\+t}} 控制接口： 主要提供 {\ttfamily read\+\_\+reg} 和 {\ttfamily write\+\_\+reg} API 来配置编解码器设备 常用控制通道包括 I2\+C， SPI 等
\item {\ttfamily \doxylink{structaudio__codec__data__if__t}{audio\+\_\+codec\+\_\+data\+\_\+if\+\_\+t}} 数据接口： 主要提供 {\ttfamily read} 和 {\ttfamily write} API 用来交换音频数据 常用数据通道包括 I2\+S， SPI 等
\end{DoxyItemize}

{\ttfamily esp\+\_\+codec\+\_\+dev} 为用户提供便捷的上层 API 来实现播放和录音功能。它是由 {\ttfamily \doxylink{structaudio__codec__data__if__t}{audio\+\_\+codec\+\_\+data\+\_\+if\+\_\+t}} 和 {\ttfamily \doxylink{structaudio__codec__if__t}{audio\+\_\+codec\+\_\+if\+\_\+t}} 组成。{\ttfamily \doxylink{structaudio__codec__if__t}{audio\+\_\+codec\+\_\+if\+\_\+t}} 对编解码器控制操作进行抽象，通过编解码器特有的配置参数构建（由 {\ttfamily \doxylink{structaudio__codec__ctrl__if__t}{audio\+\_\+codec\+\_\+ctrl\+\_\+if\+\_\+t}} 和{\ttfamily \doxylink{structaudio__codec__gpio__if__t}{audio\+\_\+codec\+\_\+gpio\+\_\+if\+\_\+t}} 通过 {\ttfamily \doxylink{structes8311__codec__cfg__t}{es8311\+\_\+codec\+\_\+cfg\+\_\+t}} 进行配置）。{\ttfamily \doxylink{structaudio__codec__gpio__if__t}{audio\+\_\+codec\+\_\+gpio\+\_\+if\+\_\+t}} 对 IO 控制进行抽象，以适配主控 IO 或者扩展芯片 IO， 在编解码器内部进行调用用以匹配特有的设定时序。\hypertarget{md_managed__components_2espressif____esp__codec__dev_2README__CN_autotoc_md253}{}\doxysection{\texorpdfstring{解码器音量设定}{解码器音量设定}}\label{md_managed__components_2espressif____esp__codec__dev_2README__CN_autotoc_md253}
音量统一通过 API {\ttfamily esp\+\_\+codec\+\_\+dev\+\_\+set\+\_\+out\+\_\+vol} 进行设定。 {\ttfamily esp\+\_\+codec\+\_\+dev} 支持以下音量设定实现\+:
\begin{DoxyEnumerate}
\item 通过调节音量寄存器实现
\item 在硬件不支持音量调节下，可以使用内置的软件音量实现 {\ttfamily audio\+\_\+codec\+\_\+new\+\_\+sw\+\_\+vol}
\item 客制化音量接口实现，并通过 {\ttfamily esp\+\_\+codec\+\_\+dev\+\_\+set\+\_\+vol\+\_\+handler} 设定
\end{DoxyEnumerate}

默认的音量调节区间是 0 -\/ 100，音量 100 对应为 0 d\+B，每个刻度对应 0.\+5 d\+B，音量 0 被特殊映射为 -\/96 d\+B。用户可以通过 {\ttfamily esp\+\_\+codec\+\_\+dev\+\_\+set\+\_\+vol\+\_\+curve} 设定音量曲线来改变这一行为。音量曲线是音量映射 {\ttfamily \doxylink{structesp__codec__dev__vol__map__t}{esp\+\_\+codec\+\_\+dev\+\_\+vol\+\_\+map\+\_\+t}} 的数组，通过线性插值的方法来计算每一个音量对应的 dB 值 (请预先将音量映射排好序)。

为了平衡在不同平台上播放相同内容的响度差异，需要了解音频增益的相关机制。简单说来音频增益包括软件增益 （可调节）和硬件增益 (不可调节) 两部分。软件增益可以通过改变音频数据的幅值或者改变音量寄存器实现。硬件增益受外围电路的影响，主要取决于模拟信号的放大系数。实现中选取了典型的影响参数 {\ttfamily \doxylink{structesp__codec__dev__hw__gain__t}{esp\+\_\+codec\+\_\+dev\+\_\+hw\+\_\+gain\+\_\+t}}，作为配置参数进行配置，以抵消平台间响度差异，详情可参考代码注释 \href{include/esp_codec_dev_vol.h}{\texttt{ esp\+\_\+codec\+\_\+dev\+\_\+vol.\+h}}。\hypertarget{md_managed__components_2espressif____esp__codec__dev_2README__CN_autotoc_md254}{}\doxysection{\texorpdfstring{使用方法}{使用方法}}\label{md_managed__components_2espressif____esp__codec__dev_2README__CN_autotoc_md254}
