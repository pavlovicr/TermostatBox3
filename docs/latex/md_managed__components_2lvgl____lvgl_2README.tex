\hypertarget{md_managed__components_2lvgl____lvgl_2README_autotoc_md486}{}\doxysubsection{\texorpdfstring{📒 Overview}{📒 Overview}}\label{md_managed__components_2lvgl____lvgl_2README_autotoc_md486}
{\bfseries{LVGL}} is a free and open-\/source UI library that enables you to create graphical user interfaces for any MCUs and MPUs from any vendor on any platform.

{\bfseries{Requirements}}\+: LVGL has no external dependencies, which makes it easy to compile for any modern target, from small MCUs to multi-\/core Linux-\/based MPUs with 3D support. For a simple UI, you need only \texorpdfstring{$\sim$}{\string~}100kB RAM, \texorpdfstring{$\sim$}{\string~}200–300kB flash, and a buffer size of 1/10 of the screen for rendering.

{\bfseries{To get started}}, pick a ready-\/to-\/use VSCode, Eclipse, or any other project and try out LVGL on your PC. The LVGL UI code is fully platform-\/independent, so you can use the same UI code on embedded targets too.

{\bfseries{LVGL Pro}} is a complete toolkit to help you build, test, share, and ship UIs faster. It comes with an XML Editor where you can quickly create and test reusable components, export C code, or load the XMLs at runtime. Learn more here.\hypertarget{md_managed__components_2lvgl____lvgl_2README_autotoc_md487}{}\doxysubsection{\texorpdfstring{💡 Features}{💡 Features}}\label{md_managed__components_2lvgl____lvgl_2README_autotoc_md487}
{\bfseries{Free and Portable}}
\begin{DoxyItemize}
\item A fully portable C (C++ compatible) library with no external dependencies.
\item Can be compiled for any MCU or MPU, with any (RT)OS. Make, CMake, and simple globbing are all supported.
\item Supports monochrome, e\+Paper, OLED, or TFT displays, or even monitors. \href{https://docs.lvgl.io/master/details/main-modules/display/index.html}{\texttt{ Displays}}
\item Distributed under the MIT license, so you can easily use it in commercial projects too.
\item Needs only 32kB RAM and 128kB Flash, a frame buffer, and at least a 1/10 screen-\/sized buffer for rendering.
\item OS, external memory, and GPU are supported but not required.
\end{DoxyItemize}

{\bfseries{Widgets, Styles, Layouts, and More}}
\begin{DoxyItemize}
\item 30+ built-\/in \href{https://docs.lvgl.io/master/details/widgets/index.html}{\texttt{ Widgets}}\+: Button, Label, Slider, Chart, Keyboard, Meter, Arc, Table, and many more.
\item Flexible \href{https://docs.lvgl.io/master/details/common-widget-features/styles/index.html}{\texttt{ Style system}} with \texorpdfstring{$\sim$}{\string~}100 style properties to customize any part of the widgets in any state.
\item \href{https://docs.lvgl.io/master/details/common-widget-features/layouts/flex.html}{\texttt{ Flexbox}} and \href{https://docs.lvgl.io/master/details/common-widget-features/layouts/grid.html}{\texttt{ Grid}}-\/like layout engines to automatically size and position the widgets responsively.
\item Text is rendered with UTF-\/8 encoding, supporting CJK, Thai, Hindi, Arabic, and Persian writing systems.
\item \href{https://docs.lvgl.io/master/details/auxiliary-modules/observer/index.html}{\texttt{ Data bindings}} to easily connect the UI with the application.
\item Rendering engine supports animations, anti-\/aliasing, opacity, smooth scrolling, shadows, image transformation, etc.
\item \href{https://docs.lvgl.io/master/details/libs/gltf.html}{\texttt{ Powerful 3D rendering engine}} to show \href{https://sketchfab.com/}{\texttt{ gl\+TF models}} with Open\+GL.
\item Supports Mouse, Touchpad, Keypad, Keyboard, External buttons, Encoder \href{https://docs.lvgl.io/master/details/main-modules/indev.html}{\texttt{ Input devices}}.
\item \href{https://docs.lvgl.io/master/details/main-modules/display/overview.html\#how-many-displays-can-lvgl-use}{\texttt{ Multiple display}} support.
\end{DoxyItemize}\hypertarget{md_managed__components_2lvgl____lvgl_2README_autotoc_md488}{}\doxysubsection{\texorpdfstring{📦️ Platform Support}{📦️ Platform Support}}\label{md_managed__components_2lvgl____lvgl_2README_autotoc_md488}
