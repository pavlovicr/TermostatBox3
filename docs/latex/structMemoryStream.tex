\doxysubsection{Podroben opis}
Represents an in-\/memory input byte stream. 

This class is mainly for being wrapped by \doxylink{classEncodedInputStream}{Encoded\+Input\+Stream} or \doxylink{classAutoUTFInputStream}{Auto\+UTFInput\+Stream}.

It is similar to File\+Read\+Buffer but the source is an in-\/memory buffer instead of a file.

Differences between \doxylink{structMemoryStream}{Memory\+Stream} and String\+Stream\+:
\begin{DoxyEnumerate}
\item String\+Stream has encoding but \doxylink{structMemoryStream}{Memory\+Stream} is a byte stream.
\item \doxylink{structMemoryStream}{Memory\+Stream} needs size of the source buffer and the buffer don\textquotesingle{}t need to be null terminated. String\+Stream assume null-\/terminated string as source.
\item \doxylink{structMemoryStream}{Memory\+Stream} supports Peek4() for encoding detection. String\+Stream is specified with an encoding so it should not have Peek4(). \begin{DoxyNote}{Opomba}
implements Stream concept 
\end{DoxyNote}

\end{DoxyEnumerate}
