\href{https://docs.espressif.com/projects/esp-idf/en/latest/esp32/api-reference/peripherals/lcd.html}{\texttt{ esp\+\_\+lcd}} supports I2C interfaced OLED LCD, whose color depth is usually 1bpp.

This example shows how to make use of the SSD1306 panel driver from {\ttfamily esp\+\_\+lcd} component to facilitate the porting of LVGL library. In the end, example will display a scrolling text on the OLED screen.\hypertarget{md_managed__components_2espressif____esp__lvgl__port_2examples_2i2c__oled_2README_autotoc_md372}{}\doxysubsection{\texorpdfstring{LVGL Version}{LVGL Version}}\label{md_managed__components_2espressif____esp__lvgl__port_2examples_2i2c__oled_2README_autotoc_md372}

Please pay attention to the I2C hardware device address as well, you should refer to your module\textquotesingle{}s spec and schematic to determine that address.\hypertarget{md_managed__components_2espressif____esp__lvgl__port_2examples_2i2c__oled_2README_autotoc_md376}{}\doxysubsubsection{\texorpdfstring{Build and Flash}{Build and Flash}}\label{md_managed__components_2espressif____esp__lvgl__port_2examples_2i2c__oled_2README_autotoc_md376}
Run {\ttfamily idf.\+py -\/p PORT build flash monitor} to build, flash and monitor the project. A scrolling text will show up on the LCD as expected.

The first time you run {\ttfamily idf.\+py} for the example will cost extra time as the build system needs to address the component dependencies and downloads the missing components from registry into {\ttfamily managed\+\_\+components} folder.

(To exit the serial monitor, type {\ttfamily Ctrl-\/\mbox{]}}.)
