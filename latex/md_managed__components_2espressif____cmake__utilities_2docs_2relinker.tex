 In ESP-\/\+IDF, some functions are put in SRAM when link stage, the reason is that some functions are critical, we need to put them in SRAM to speed up the program, or the functions will be executed when the cache is disabled. But actually, some functions can be put into Flash, here, we provide a script to let the user set the functions which are located in SRAM by default to put them into Flash, in order to save more SRAM which can be used as heap region later. This happens in the linker stage, so we call it as relinker.\hypertarget{md_managed__components_2espressif____cmake__utilities_2docs_2relinker_autotoc_md98}{}\doxysection{\texorpdfstring{Use}{Use}}\label{md_managed__components_2espressif____cmake__utilities_2docs_2relinker_autotoc_md98}
In order to use this feature, you need to include the needed CMake file in your project\textquotesingle{}s CMake\+Lists.\+txt after {\ttfamily project(\+XXXX)}.


\begin{DoxyCode}{0}
\DoxyCodeLine{project(XXXX)}
\DoxyCodeLine{}
\DoxyCodeLine{include(relinker)}

\end{DoxyCode}


The relinker feature is disabled by default, in order to use it, you need to enable the option {\ttfamily CU\+\_\+\+RELINKER\+\_\+\+ENABLE} in menuconfig.

Here are the default configuration files in the folder {\ttfamily cmake\+\_\+utilities/scripts/relinker/examples/esp32c2}, it\textquotesingle{}s just used as a reference. If you would like to use your own configuration files, please enable option {\ttfamily CU\+\_\+\+RELINKER\+\_\+\+ENABLE\+\_\+\+CUSTOMIZED\+\_\+\+CONFIGURATION\+\_\+\+FILES} and set the path of your configuration files as following, this path is evaluated relative to the project root directory\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{[*]\ \ \ \ \ Enable\ customized\ relinker\ configuration\ files}
\DoxyCodeLine{(path\ of\ your\ configuration\ files)\ Customized\ relinker\ configuration\ files\ path}

\end{DoxyCode}


\begin{quote}
Note\+: Currently only esp32c2 is supported. \end{quote}
\hypertarget{md_managed__components_2espressif____cmake__utilities_2docs_2relinker_autotoc_md99}{}\doxysection{\texorpdfstring{Configuration Files}{Configuration Files}}\label{md_managed__components_2espressif____cmake__utilities_2docs_2relinker_autotoc_md99}
You can refer to the files in the directory of {\ttfamily cmake\+\_\+utilities/scripts/relinker/examples/esp32c2}\+:


\begin{DoxyItemize}
\item library.\+csv
\item object.\+csv
\item function.\+csv
\end{DoxyItemize}

For example, if you want to link function {\ttfamily \+\_\+\+\_\+getreent} from SRAM to Flash, firstly you should add it to {\ttfamily function.\+csv} file as following\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{libfreertos.a,tasks.c.obj,\_\_getreent,}

\end{DoxyCode}


This means function {\ttfamily \+\_\+\+\_\+getreent} is in object file {\ttfamily tasks.\+c.\+obj}, and object file {\ttfamily tasks.\+c.\+obj} is in library {\ttfamily libfreertos.\+a}.

If function {\ttfamily \+\_\+\+\_\+getreent} depends on the option {\ttfamily FREERTOS\+\_\+\+PLACE\+\_\+\+FUNCTIONS\+\_\+\+INTO\+\_\+\+FLASH} in menuconfig, then it should be\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{libfreertos.a,tasks.c.obj,\_\_getreent,CONFIG\_FREERTOS\_PLACE\_FUNCTIONS\_INTO\_FLASH}

\end{DoxyCode}


This means when only {\ttfamily FREERTOS\+\_\+\+PLACE\+\_\+\+FUNCTIONS\+\_\+\+INTO\+\_\+\+FLASH} is enabled in menuconfig, function {\ttfamily \+\_\+\+\_\+getreent} will be linked from SRAM to Flash.

Next step you should add the path of the object file to {\ttfamily object.\+csv}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{libfreertos.a,tasks.c.obj,esp-\/idf/freertos/CMakeFiles/\_\_idf\_freertos.dir/FreeRTOS-\/Kernel/tasks.c.obj}

\end{DoxyCode}


This means the object file {\ttfamily tasks.\+c.\+obj} is in library {\ttfamily libfreertos.\+a} and its location is {\ttfamily esp-\/idf/freertos/\+CMake\+Files/\+\_\+\+\_\+idf\+\_\+freertos.\+dir/\+Free\+RTOS-\/\+Kernel/tasks.c.\+obj} relative to directory of {\ttfamily build}.

Next step you should add path of library to {\ttfamily library.\+csv}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{libfreertos.a,./esp-\/idf/freertos/libfreertos.a}

\end{DoxyCode}


This means library {\ttfamily libfreertos.\+a}\textquotesingle{}s location is {\ttfamily ./esp-\/idf/freertos/libfreertos.a} relative to {\ttfamily build}.

If above related data has exists in corresponding files, please don\textquotesingle{}t add this repeatedly.
