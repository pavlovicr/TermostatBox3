After creating a new button object by calling function {\ttfamily button\+\_\+create()}, the button object can create press events, every press event can have its own callback.

List of supported events\+:
\begin{DoxyItemize}
\item Button pressed
\item Button released
\item Button pressed repeat
\item Button press repeat done
\item Button single click
\item Button double click
\item Button multiple click
\item Button long press start
\item Button long press hold
\item Button long press up
\item Button Press end
\end{DoxyItemize}



There are three ways this driver can handle buttons\+:
\begin{DoxyEnumerate}
\item Buttons connected to standard digital GPIO
\item Multiple buttons connected to single ADC channel
\item Matrix keyboard employs multiple GPIOs for operation.
\item Custom button connect to any driver
\end{DoxyEnumerate}

The component supports the following functionalities\+:
\begin{DoxyEnumerate}
\item Creation of an unlimited number of buttons, accommodating various types simultaneously.
\item Multiple callback functions for a single event.
\item Allowing customization of the consecutive key press count to any desired number.
\item Facilitating the setup of callbacks for any specified long-\/press duration.
\item Support power save mode (Only for gpio button)
\end{DoxyEnumerate}\hypertarget{md_managed__components_2espressif____button_2README_autotoc_md68}{}\doxysubsection{\texorpdfstring{Add component to your project}{Add component to your project}}\label{md_managed__components_2espressif____button_2README_autotoc_md68}
